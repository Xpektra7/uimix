## Customization

### Updating Workflow Content

Modify the `flows` array to customize your workflow stages:

```tsx
const flows = [
  {
    id: "01",
    variant: "orbit",       // Icon animation type
    meta: "Discovery",      // Category tag
    title: "Signal Mapping Sprints",
    description: "Frame the opportunity space...",
    statLabel: "Scope commit",
    statValue: "48 hrs",
  },
  // Add more workflow stages...
];
```

### Metrics Configuration

Update the metrics displayed in the grid:

```tsx
const metrics = [
  { label: "Mean delivery", value: "19 days" },
  { label: "Release confidence", value: "99.5%" },
  { label: "Audited rollouts", value: "120+" },
];
```

### Animation Timings

All animations are CSS-based and can be customized in the injected stylesheet:

```css
@keyframes bento3-card-in {
  0% { opacity: 0; transform: translate3d(0, 28px, 0) scale(0.97); 
       filter: blur(12px); }
  60% { filter: blur(0); }
  100% { opacity: 1; transform: translate3d(0, 0, 0) scale(1); 
         filter: blur(0); }
}
```

## Theme System

### Color Palettes

The component includes two complete color palettes:

**Dark Palette:**
- Surface: `bg-neutral-950 text-neutral-100`
- Cards: `bg-neutral-900/55 border-white/10`
- Accents: White with various opacity levels

**Light Palette:**
- Surface: `bg-slate-100 text-neutral-900`
- Cards: `bg-white/80 border-neutral-200`
- Accents: Neutral tones with controlled contrast

### Theme Detection Priority

1. Document `classList` (dark/light)
2. `data-theme` attribute
3. localStorage (`bento-theme`)
4. System preference (`prefers-color-scheme`)
5. Cross-tab sync via Storage events

## Hover Effects

Cards feature dynamic radial glow effects that follow the cursor:

```tsx
const setGlow = (event: React.MouseEvent) => {
  const rect = target.getBoundingClientRect();
  target.style.setProperty("--bento3-x", `${event.clientX - rect.left}px`);
  target.style.setProperty("--bento3-y", `${event.clientY - rect.top}px`);
};
```

The glow is rendered as a radial gradient positioned at cursor coordinates.

## Intersection Observer

Cards animate in when scrolled into view:

```tsx
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        setVisible(true);
        observer.disconnect();
      }
    });
  },
  { threshold: 0.2 }
);
```

## Performance Notes

- All animations are CSS-based (no JavaScript loops)
- Intersection Observer prevents unnecessary renders
- Styles injected once and cleaned up on unmount
- Hardware-accelerated transforms (translate3d)
- Staggered animations use CSS `animation-delay`

## Accessibility

- Semantic HTML with proper landmarks
- Theme toggle has `aria-pressed` state
- Focus states on interactive elements
- Respects motion preferences (can be enhanced)
